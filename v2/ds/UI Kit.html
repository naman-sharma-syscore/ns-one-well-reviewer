00 — Principles

What this system is for

One WELL helps real estate, workplace, community, and product teams pursue and verify health-focused certifications. The system carries that mission into the interface — calm, factual, generous, and never alarmist.

Five working principles
  • Restraint over flourish. One primary CTA per view. Concept colors only on tags. FT Made only for editorial display.
  • Brand marks are sacred. Seals never recolored, redrawn, or stylized. Use them verbatim.
  • Tokens, not values. Every color, size, font reference is a CSS variable. No hex codes inline.
  • One canonical home. Each component has a single preview card. Other places link to it; never re-define.
  • Generous, never urgent. "8 points to Gold," not "Hurry — only 8 points!"
Five anti-patterns
  • Hardcoded hex codes anywhere
  • Recolored or redrawn seals/badges
  • Multiple primary CTAs on one screen
  • Concept colors on chrome (buttons, panels, headers)
  • Decorative emoji, urgency-bait copy, "Oops!" error language
For AI design agents CLAUDE.md is auto-loaded into every prototype session It contains the full token list, component contracts, voice rules, and anti-patterns. Anyone designing with Claude in this project automatically follows the system.

08 — Guidelines

Voice, accessibility, dos & don'ts

The full guideline document is in CLAUDE.md — auto-loaded by Claude in every design session. The condensed dos/don'ts below cover the most-flagged review issues.

✓ Do
  • Link colors_and_type.css + components.css at the top of every HTML file
  • Use .ow-btn-* classes — they bake in optical alignment
  • One primary CTA per view. Identify the action the user is here to complete.
  • Use side-rail pattern (--cyan-50) only for nav/rails — never as content background
  • Reach for seals at ≥64px; below that, swap to badge variant
  • Sentence case for headings; Title Case only for proper nouns + cert levels
  • Errors as action: "We couldn't reach the reviewer — try again in a minute"
  • Body contrast ≥4.5:1; touch targets ≥44×44px
✗ Don't
  • Hardcoded hex codes anywhere — always var(--token)
  • Recolor or redraw seals/badges — they are sacred
  • Multiple primary CTAs in one view
  • Concept colors on chrome (buttons, panels, headers) — tagging only
  • FT Made on UI controls (buttons, labels, fields) — display only
  • Performance gradient as page background — portfolio rating bars only
  • Decorative emoji or "Oops!" / "Hurry!" copy
  • outline:none without a custom focus replacement
